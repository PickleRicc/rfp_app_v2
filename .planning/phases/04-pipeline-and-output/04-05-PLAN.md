<task type="auto">
  <name>Task 1: Integrate PageTracker into stage2-response-generator</name>
  <files>lib/inngest/functions/stage2-response-generator.ts</files>
  <action>
Update the stage2 response generator to integrate page tracking:

1. Import PageTracker from lib/generation/pipeline
2. Create PageTracker instance with page limits from RFP analysis (if available)
3. Track pages as each section is generated
4. Log warnings when approaching limits
5. Store page allocation summary in proposal metadata

Key integration points:
- After each volume generation, call pageTracker.addSection()
- Before finalizing, check pageTracker.getStatus()
- If status is 'over', trigger condense callback (optional - can be manual for v1)
- Store allocation summary in proposal record for display

Do NOT restructure the existing generator - add tracking as a parallel concern.

Example integration:
```typescript
import { PageTracker } from '@/lib/generation/pipeline';

// In the generation function:
const pageTracker = new PageTracker();

// Set limits from RFP if available
if (rfpAnalysis.pageLimits) {
  pageTracker.setVolumeLimits(rfpAnalysis.pageLimits);
}

// After generating technical volume:
pageTracker.addSection({
  sectionId: 'technical',
  volumeType: 'Technical',
  content: technicalContent,
});

// At end, get status:
const pageStatus = pageTracker.getStatus();
if (pageStatus.status === 'over') {
  console.warn(`Page limit exceeded: ${pageStatus.message}`);
}

// Store in proposal:
await supabase.from('proposals').update({
  page_allocation: pageStatus
}).eq('id', proposalId);
```

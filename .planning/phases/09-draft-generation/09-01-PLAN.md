  <name>Task 1: Database schema and TypeScript types for draft generation</name>
  <files>
    supabase_tables/supabase-draft-generation-migration.sql

  </files>
  <action>
    Create SQL migration with two tables:

    **`proposal_drafts`** — one row per solicitation + company draft generation run:
    - `id` UUID PK DEFAULT gen_random_uuid()
    - `solicitation_id` UUID FK references solicitations(id) ON DELETE CASCADE
    - `company_id` UUID FK references company_profiles(id) ON DELETE CASCADE
    - `status` TEXT CHECK (pending | generating | completed | failed)
    - `strategy_confirmed_at` TIMESTAMPTZ — when user clicked Confirm & Generate
    - `generation_started_at` TIMESTAMPTZ
    - `generation_completed_at` TIMESTAMPTZ
    - `error_message` TEXT — null unless failed
    - `created_at` / `updated_at` TIMESTAMPTZ defaults
    - UNIQUE (solicitation_id, company_id) — one active draft per solicitation per company (regeneration overwrites)
    - RLS enabled with authenticated policy (IF NOT EXISTS pattern matching Phase 8)
    - Index on (solicitation_id, company_id)

    **`draft_volumes`** — one row per generated volume within a draft:
    - `id` UUID PK DEFAULT gen_random_uuid()
    - `draft_id` UUID FK references proposal_drafts(id) ON DELETE CASCADE
    - `volume_name` TEXT NOT NULL — matches Section L volume name (e.g., "Technical Approach", "Management", "Past Performance", "Cost/Price")
    - `volume_order` INT NOT NULL — display order from Section L
    - `status` TEXT CHECK (pending | generating | completed | failed)
    - `word_count` INT — actual word count of generated content
    - `page_estimate` NUMERIC(5,1) — estimated pages via heuristic
    - `page_limit` INT — RFP page limit for this volume (from Section L extraction)
    - `page_limit_status` TEXT CHECK (under | warning | over) — color-coded status
    - `file_path` TEXT — Supabase storage path to generated DOCX
    - `content_markdown` TEXT — raw markdown content before DOCX conversion (for in-app preview)
    - `error_message` TEXT
    - `generation_started_at` / `generation_completed_at` TIMESTAMPTZ
    - `created_at` / `updated_at` TIMESTAMPTZ defaults
    - RLS enabled with authenticated policy
    - Index on (draft_id, volume_order)

    Create TypeScript types file `lib/supabase/draft-types.ts` following the pattern of tier2-types.ts:
    - `DraftStatus` = 'pending' | 'generating' | 'completed' | 'failed'
    - `VolumeStatus` = 'pending' | 'generating' | 'completed' | 'failed'
    - `PageLimitStatus` = 'under' | 'warning' | 'over'
    - `ProposalDraft` interface mirroring the table
    - `DraftVolume` interface mirroring the table
    - `StrategyConfirmation` interface — the assembled read-only summary:
      - company_name, legal_name, uei_number, cage_code (from Tier 1)
      - prime_or_sub, teaming_partners, contract_type, naics_code, set_aside (from Tier 2 opportunity_details)
      - enterprise_win_themes, key_differentiators_summary (from Tier 1)
      - volume_structure (from Section L extraction) — array of { name, page_limit }
      - evaluation_factors (from Section M extraction) — array of { name, weight }
      - key_personnel (from Tier 2) — array of { role, name }
      - past_performance_count (from Tier 2)
    - `DraftGetResponse` = { draft: ProposalDraft | null, volumes: DraftVolume[], strategy: StrategyConfirmation }
    - Display constants: `DRAFT_STATUS_LABELS`, `DRAFT_STATUS_COLORS`, `PAGE_LIMIT_STATUS_COLORS`
  </action>

<task type="auto">
  <name>Task 2: Strategy confirmation, draft trigger, status, and download API endpoints</name>
  <files>
    app/api/solicitations/[id]/draft/route.ts
    app/api/solicitations/[id]/draft/volumes/[volumeId]/route.ts

  <action>
    Create `app/api/solicitations/[id]/draft/route.ts` with GET and POST handlers following the data-call/route.ts auth pattern (requireStaffOrResponse + X-Company-Id + solicitation ownership check + async params for Next.js 15+):

    **GET** — Returns strategy confirmation + draft status + volumes:
    1. Auth + ownership check
    2. Assemble `StrategyConfirmation` by querying:
       - `company_profiles` (Tier 1 fields: company_name, legal_name, uei_number, cage_code, enterprise_win_themes, key_differentiators_summary)
       - `data_call_responses` (Tier 2: opportunity_details, past_performance count, key_personnel roles/names)
       - `compliance_extractions` where category='section_l' (volume_structure, page_limits) and category='section_m' (evaluation_factors, weightings)
    3. Fetch existing `proposal_drafts` row (if any) + associated `draft_volumes` ordered by volume_order
    4. Return `{ strategy, draft, volumes }` as DraftGetResponse

    **POST** — Triggers draft generation:
    1. Auth + ownership check
    2. Verify data_call_responses.status = 'completed' for this solicitation — reject with 400 "Data call must be completed before generating draft" if not
    3. Upsert proposal_drafts: set status='pending', strategy_confirmed_at=now(), clear old error_message
    4. Delete existing draft_volumes rows (regeneration per CONTEXT.md decision — overwrites old draft)
    5. Create draft_volumes rows in 'pending' status based on volume_structure from Section L extraction (query compliance_extractions where field_name='volume_structure'). For each volume: insert { draft_id, volume_name, volume_order, status: 'pending', page_limit }
    6. Send Inngest event: `{ name: 'proposal.draft.generate', data: { solicitationId, companyId, draftId } }`
    7. Return 200 with { draft, volumes, message: "Draft generation started" }

    Create `app/api/solicitations/[id]/draft/volumes/[volumeId]/route.ts`:

    **GET** — Serves DOCX file for download:
    1. Auth + ownership check (join draft_volumes → proposal_drafts to verify company ownership)
    2. Fetch draft_volumes row by volumeId
    3. If status !== 'completed' or file_path is null, return 404
    4. Download DOCX from Supabase storage using file_path
    5. Return binary response with Content-Type application/vnd.openxmlformats-officedocument.wordprocessingml.document and Content-Disposition attachment; filename="{volume_name}.docx"
  </action>

  </verify>

    - "Confirm & Generate" — primary button (blue), calls POST `/api/solicitations/{id}/draft`, on success calls `onGenerationStarted()` to switch to generation view. Show Loader2 spinner while POST is in-flight. Disable button if already generating (draft.status === 'generating').

    Style: Use Tailwind classes matching the existing DataCallView card patterns. Read-only fields displayed as text (not inputs). Clean, professional layout.

    **Create `app/components/draft/DraftGenerationView.tsx`:**

    Props: `{ solicitationId: string, companyId: string }`

    This is the main draft view component that manages the full flow:

    State machine with three views:
    1. **strategy** — shows StrategyConfirmation component (default when no draft exists or draft.status === 'pending')
    2. **generating** — shows generation progress (when draft.status === 'generating')
    3. **complete** — shows volume list with preview/download (when draft.status === 'completed' or 'failed')

    On mount: fetch GET `/api/solicitations/{id}/draft`. Based on draft.status, set initial view.

    **Generating view:**
    - Header: "Generating Draft..." with animated Loader2 icon
    - Volume checklist: ordered list of volumes, each showing:
      - Volume name
      - Status icon: Loader2 spinning (generating), CheckCircle green (completed), XCircle red (failed), Clock gray (pending)
      - Status text
    - Poll GET `/api/solicitations/{id}/draft` every 3 seconds (setInterval). Stop polling when draft.status is 'completed' or 'failed'. Clear interval on unmount.
    - When all done, auto-transition to complete view.

    **Complete view:**
    - Header: "Draft Generated" with CheckCircle or "Draft Generation Complete (with issues)" if some volumes failed
    - **Summary bar**: horizontal row showing total volumes completed, total word count, overall status
    - **Volume cards** (one per draft_volume, ordered by volume_order, excluding compliance matrix):
      Each card has:
      - Volume name as heading
      - Page limit status bar: horizontal bar showing page_estimate / page_limit ratio
        - Green fill when page_limit_status='under' (< 85%)
        - Yellow fill when page_limit_status='warning' (85-100%)
        - Red fill when page_limit_status='over' (> 100%)
        - Text: "{page_estimate} / {page_limit} pages ({word_count} words)"
        - If no page_limit: just show "{page_estimate} pages ({word_count} words)" with no bar
      - Expandable preview section: click "Preview" to expand/collapse. Shows content_markdown rendered as formatted text (use a simple markdown-to-JSX approach: split by ### for headings, ** for bold, newlines for paragraphs). Keep it simple — no full markdown parser needed. Render as styled divs with Tailwind prose classes if available, otherwise manual formatting.
      - "Download DOCX" button: links to `/api/solicitations/{id}/draft/volumes/{volumeId}` (opens in new tab or triggers download)
      - If status='failed': show red error badge with error_message, no preview/download

    - **Compliance Matrix section** (after volume cards):
      - Find the draft_volume with volume_name="Compliance Matrix"
      - Render its content_markdown as a formatted table (markdown table → HTML table)
      - Header: "Compliance Matrix — Evaluation Factor Coverage"

    - **"Regenerate Draft" button** at bottom: triggers POST, transitions back to strategy view or directly to generating view

    Polling cleanup: useEffect returns clearInterval.

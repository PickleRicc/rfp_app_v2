<objective>
Build the Inngest draft generation pipeline that produces compliant proposal volumes. The pipeline fetches all data sources (Tier 1 corporate profile, Tier 2 data call, compliance extractions), assembles volume-specific AI prompts, calls Claude to generate content for each volume, converts content to DOCX using the existing v1.0 generator, and stores results. Each volume targets 85-90% of its RFP page limit. A compliance matrix maps Section M evaluation factors to generated content locations.

Purpose: This is the core generation engine — transforms customer and RFP data into professional proposal volumes ready for download.
Output: Prompt assembler module, Inngest pipeline function, updated Inngest route registration.
</objective>

  <name>Task 1: Create prompt assembler for volume-specific AI content generation</name>
  <files>lib/generation/draft/prompt-assembler.ts</files>
  <action>
    Create `lib/generation/draft/prompt-assembler.ts` — the module that transforms raw data from all three sources into structured AI prompts tailored to each volume type.

    **Core function: `assembleVolumePrompt(volumeName, volumeOrder, data)`**

    Parameters:
    - `volumeName`: string — the Section L volume name (e.g., "Technical Approach")
    - `volumeOrder`: number — position in the volume sequence
    - `data`: object containing:
      - `companyProfile`: CompanyProfile (Tier 1)
      - `dataCallResponse`: DataCallResponse (Tier 2)
      - `extractions`: ComplianceExtractionsByCategory (from Phase 7)
      - `pageLimit`: number | null
      - `pageLimitSpacing`: 'single' | 'double' (default 'single')

    Returns: `{ systemPrompt: string, userPrompt: string, targetWordCount: number }`

    **System prompt** (shared across all volumes):
    ```
    You are an expert government proposal writer with 20+ years of experience winning federal contracts. You write in a formal, compliance-focused style using action verbs and "shall/will" phrasing. Every claim is backed by specific evidence. You mirror the evaluation criteria language from the RFP. You are writing for {companyProfile.company_name} ({companyProfile.legal_name}).
    ```

    **Volume-specific user prompts** — detect volume type from volumeName using keyword matching (case-insensitive):

    1. **Technical volume** (matches "technical", "approach"):
       - Include: company core_services_summary, technical_approach (tools_and_platforms, staffing_model, approach_summary) from Tier 2
       - Include: SOW/PWS task areas from sow_pws extraction category
       - Include: evaluation factors from Section M that relate to technical merit
       - Instruct: Address each technical evaluation subfactor explicitly, use requirements traceability boxes per the Proposal_Response_Generation_Framework
       - Instruct: Include win theme callout boxes at start of major sections using enterprise_win_themes from Tier 1

    2. **Management volume** (matches "management", "staffing", "transition"):
       - Include: standard_management_approach from Tier 1
       - Include: key_personnel from Tier 2 (roles, names, qualifications)
       - Include: organizational structure from Tier 1 corporate_overview
       - Instruct: Staffing plan section, transition plan (in/out), quality control plan, risk management
       - Instruct: Win theme callout boxes

    3. **Past performance volume** (matches "past perf"):
       - Include: past_performance array from Tier 2 (all references)
       - Include: relevance criteria from Section M past_performance extraction
       - Include: recency/relevance definitions from rating_scales extraction
       - Instruct: One section per reference, structured as: contract overview, relevance to current requirement, outcomes/achievements, relevance rating justification

    4. **Cost/Price volume** (matches "cost", "price"):
       - Include: contract_type from Tier 2 opportunity_details
       - Include: CLIN structure from admin_data extraction
       - Include: period of performance from admin_data
       - Instruct: Price narrative / basis of estimate structure, NOT actual pricing numbers (user fills those in Word)
       - Use [PLACEHOLDER] markers for all dollar amounts and rates

    5. **Default** (any unmatched volume name):
       - Include all available data
       - Instruct: Address the volume name as a topic, follow Section L instructions

    **Page limit targeting:**
    - If pageLimit is provided, calculate targetWordCount = pageLimit * wordsPerPage * 0.875 (midpoint of 85-90%)
    - wordsPerPage = 250 for single-spaced, 500 for double-spaced (default single per gov RFP norms)
    - Include in user prompt: "Target approximately {targetWordCount} words ({pageLimit} page limit, targeting 87.5% utilization)."
    - If no pageLimit, set targetWordCount = 2500 (default ~10 pages)

    **Win theme integration:**
    - For each enterprise_win_theme from Tier 1, instruct AI to create a callout box:
      ```
      Insert a win theme callout box at the beginning of each major section. Format as:
      === WIN THEME: {theme} ===
      {One sentence connecting theme to this section's content}
      ========================
      ```

    **Sparse data handling:**
    - If any Tier 2 field is empty/null, instruct: "Where specific data is unavailable, insert [PLACEHOLDER: describe what information is needed] so the user can fill it in during review."

    **Second function: `assembleComplianceMatrixData(extractions, volumes)`**
    - Takes Section M extractions (evaluation_factors with subfactors and weights) and the list of generated volume names
    - Returns structured data: array of { factor, subfactors, weight, mapped_volume, mapped_sections[] }
    - Maps each evaluation factor to the most relevant volume based on factor name keywords
    - This data will be used by the Inngest pipeline to generate the compliance matrix

    Export both functions plus the `VolumePromptData` and `VolumePromptResult` interfaces.

  <name>Task 2: Create Inngest draft generation pipeline and register it</name>
  <files>
    lib/inngest/functions/proposal-draft-generator.ts

  <action>
    Create `lib/inngest/functions/proposal-draft-generator.ts` following the pattern of `solicitation-compliance-extractor.ts` (Inngest steps with error isolation, processing_logs, per-step status updates).

    **Function definition:**
    ```typescript
    export const proposalDraftGenerator = inngest.createFunction(
      { id: 'proposal-draft-generator' },
      { event: 'proposal.draft.generate' },
      async ({ event, step }) => { ... }
    );
    ```

    **Event data:** `{ solicitationId: string, companyId: string, draftId: string }`

    **Step 1: `fetch-all-data`**
    - Update proposal_drafts status to 'generating', set generation_started_at
    - Fetch company_profiles row (Tier 1) by companyId
    - Fetch data_call_responses row (Tier 2) by solicitationId + companyId
    - Fetch compliance_extractions grouped by category (reuse the GET compliance pattern)
    - Fetch draft_volumes rows ordered by volume_order
    - Log to processing_logs: stage='draft-generator', status='started'
    - Return all fetched data

    **Steps 2..N: `generate-volume-{order}` (one step per volume)**
    - For each draft_volume row (in volume_order):
      1. Update draft_volumes.status = 'generating', generation_started_at = now()
      2. Call `assembleVolumePrompt(volume.volume_name, volume.volume_order, data)`
      3. Call Anthropic Claude (claude-sonnet-4-5, max_tokens 8192, temperature 0.3):
         - System message: result.systemPrompt
         - User message: result.userPrompt
      4. Extract markdown content from Claude response
      5. Calculate word count: `content.split(/\s+/).length`
      6. Calculate page estimate: `wordCount / 250` (single-spaced heuristic)
      7. Determine page_limit_status:
         - If no page_limit: 'under'
         - If pageEstimate < pageLimit * 0.85: 'under' (green)
         - If pageEstimate <= pageLimit: 'warning' (yellow)
         - If pageEstimate > pageLimit: 'over' (red)
      8. Generate DOCX using existing `generateProposalVolume()` from lib/generation/docx/generator.ts:
         - Convert markdown sections to the content structure expected by the generator
         - Pass companyProfile, exhibits=[] (no exhibits in v2.0 draft), volumeType derived from volume name
         - Call `saveDocxToBuffer()` to get the Buffer
      9. Upload DOCX buffer to Supabase storage at path: `drafts/{companyId}/{solicitationId}/{volume_order}_{sanitized_volume_name}.docx`
      10. Update draft_volumes: status='completed', word_count, page_estimate, page_limit_status, file_path, content_markdown, generation_completed_at
      11. On error in any sub-step: update draft_volumes status='failed', error_message, log error

    Each volume step is wrapped in try/catch. A single volume failure does NOT prevent other volumes from generating (error isolation pattern from Phase 7).

    **Step N+1: `generate-compliance-matrix`**
    - Call `assembleComplianceMatrixData(extractions, volumes)` from prompt-assembler
    - Generate a simple markdown compliance matrix table showing:
      - Evaluation Factor | Weight | Volume | Status
    - For each factor, status = "Addressed" if the mapped volume completed successfully, "Pending" if generating, "Gap" if volume failed
    - Store as a separate draft_volumes row with volume_name="Compliance Matrix", volume_order=999
    - This is NOT a DOCX — store content_markdown only (no file_path). The UI renders it directly.

    **Step N+2: `finalize`**
    - Count completed vs failed volumes
    - If all volumes completed: update proposal_drafts status='completed', generation_completed_at
    - If any failed but some completed: still set status='completed' (partial success — user can see which volumes need regeneration)
    - If ALL volumes failed: set status='failed', error_message summarizing failures
    - Log to processing_logs: stage='draft-generator', status='completed'/'failed', metadata with counts

    **Register the function** in `app/api/inngest/route.ts`:
    - Add import: `import { proposalDraftGenerator } from '@/lib/inngest/functions/proposal-draft-generator';`
    - Add to functions array: `proposalDraftGenerator`

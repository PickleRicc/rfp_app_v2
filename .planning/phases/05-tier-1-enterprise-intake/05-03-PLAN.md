    **lib/validation/tier1-completeness.ts** — Create completeness calculator:

    Import `CompanyProfile` from `@/lib/supabase/company-types`.
    Import `validateUEI`, `validateCAGE`, `validateWordCount` from `@/lib/validation/tier1-validators`.

    **calculateTier1Completeness(profile: Partial&lt;CompanyProfile&gt;): Tier1Completeness**
    Return object with:
    - `score`: number 0-100 (percentage complete)
    - `sections`: array of {name, score, maxScore, missingFields}
    - `isComplete`: boolean (true when all REQUIRED fields are filled and valid)

    Scoring breakdown (total 100 points):
    - Corporate Identity (35 points):
      - legal_name present and non-empty: 5
      - company_name present and non-empty: 5
      - uei_number present AND passes validateUEI(): 5
      - cage_code present AND passes validateCAGE(): 5
      - sam_status is 'Active': 5
      - sam_expiration is set: 3
      - business_size is set: 4
      - socioeconomic_certs has at least 1 entry OR business_size is 'Large' (large businesses typically don't have SB certs): 3

    - Vehicles & Certifications (25 points):
      - iso_cmmi_status has at least one true value OR all false is acceptable (some companies don't have these): 5 (always award — this is informational, not gatekeeping)
      - dcaa_approved_systems has at least one field set: 5 (always award — same reasoning)
      - Facility clearance section is filled (has_facility_clearance set to true or false explicitly): 5
      - At least 1 contract vehicle exists (check via separate query or parameter): 5
      - At least 1 NAICS code exists: 5

    - Capabilities & Positioning (40 points):
      - corporate_overview present and passes validateWordCount(text, 50, 500): 8
      - core_services_summary present and validateWordCount(text, 30, 300): 8
      - enterprise_win_themes has >= 3 entries: 8
      - key_differentiators_summary present and validateWordCount(text, 30, 300): 8
      - standard_management_approach present and validateWordCount(text, 50, 500): 8

    **getTier1MissingFields(profile: Partial&lt;CompanyProfile&gt;): string[]**
    Return human-readable list of missing/invalid fields (e.g., "UEI Number is invalid", "Corporate Overview needs at least 50 words").

    **isTier1Complete(profile: Partial&lt;CompanyProfile&gt;): boolean**
    Returns true ONLY when score >= 80 AND all Corporate Identity required fields are valid (UEI, CAGE, SAM Active, business_size set). The 80 threshold allows some optional fields to remain unfilled while ensuring core data quality.

    Export the `Tier1Completeness` type interface.

    **app/api/upload/route.ts** — Add Tier 1 gate check:
    - After existing auth and company ID resolution, BEFORE processing the upload:
    - Query `company_profiles` for the current company's `tier1_complete` field
    - If `tier1_complete` is false or null: return 403 with JSON error: {error: "Tier 1 Enterprise Intake must be completed before uploading RFP documents. Please complete your company profile at /company/profile.", code: "TIER1_INCOMPLETE"}
    - If `tier1_complete` is true: proceed with existing upload logic unchanged
    - This is a single query check (SELECT tier1_complete FROM company_profiles WHERE id = companyId), not a full completeness recalculation — the flag is set by the profile save flow

    **app/api/company/profile/route.ts** — Auto-update tier1_complete flag on PUT:
    - After successfully updating the profile, recalculate completeness:
    - Import `isTier1Complete` from `@/lib/validation/tier1-completeness`
    - Fetch the updated profile (already returned from the update query)
    - Also fetch contract_vehicles count and naics_codes count for the company
    - Call `isTier1Complete(updatedProfile)` with additional data
    - If the result differs from the current `tier1_complete` value, update the flag:
      `UPDATE company_profiles SET tier1_complete = {result} WHERE id = companyId`
    - This means the flag auto-updates whenever the user saves any tab — they don't need to manually "mark complete"
    - Return the updated completeness status in the response: `{success: true, profile: {...}, tier1Complete: boolean}`

    **Important:** The gate check on upload uses the pre-computed `tier1_complete` boolean flag for performance (single column check, no joins). The flag is kept in sync by the profile PUT handler after every save. This avoids expensive completeness recalculation on every upload request.
  </action>

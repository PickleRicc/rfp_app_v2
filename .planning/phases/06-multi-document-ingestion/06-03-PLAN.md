**lib/ingestion/reconciliation-engine.ts:**

Create the core reconciliation logic as pure functions that use Anthropic for AI analysis.

Export two main functions:

```typescript
/**
 * Compares amendment documents against the base RFP and earlier amendments.
 * Identifies sections where the amendment supersedes or modifies the base text.
 * Returns an array of reconciliation records to insert into document_reconciliations.
 */
export async function reconcileAmendments(
  baseDocument: { id: string; extracted_text: string; document_type: string },
  amendments: { id: string; extracted_text: string; amendment_number: string; effective_date?: string }[]
): Promise<ReconciliationResult[]>
```

Implementation:
- Sort amendments by amendment_number (chronological order)
- For each amendment, call Anthropic with a prompt that:
  - Provides the base document text (first 20000 chars) and the amendment text (first 15000 chars)
  - Asks Claude to identify all sections where the amendment changes the base document
  - For each change: extract the original text, the replacement text, the section reference (e.g., "Section L.4.2"), and the change_type (supersedes_section, adds_requirement, removes_requirement, general_modification)
  - Return JSON array of changes
- For multi-amendment chains: each amendment is compared against the "current truth" (base + earlier amendments), building the chain: Original -> Amend 1 -> Amend 2 (per user decision)
- Return array of ReconciliationResult objects (matching document_reconciliations schema)

```typescript
/**
 * Parses Q&A response documents to identify answers that modify requirements.
 * Looks specifically for changes to: scope, page limits, evaluation criteria, submission instructions.
 * Tags each modification as 'qa_response' source type for provenance.
 */
export async function parseQAModifications(
  qaDocument: { id: string; extracted_text: string },
  baseDocument: { id: string; extracted_text: string }
): Promise<ReconciliationResult[]>
```

Implementation:
- Call Anthropic with the Q&A text and base document text
- Prompt should specifically ask Claude to identify Q&A answers that:
  - Modify scope of work (change_type: 'modifies_scope')
  - Change page limits (change_type: 'modifies_page_limit')
  - Alter evaluation criteria (change_type: 'modifies_eval_criteria')
  - Update submission instructions (change_type: 'modifies_submission_instructions')
  - Add new requirements (change_type: 'adds_requirement')
- Each result tagged with source_type: 'qa_response' for provenance tracking (per user decision: "Q&A responses tagged as 'Q&A-sourced' for clear provenance")
- Return array of ReconciliationResult objects

Define the ReconciliationResult interface:
```typescript
interface ReconciliationResult {
  source_document_id: string;
  target_document_id: string | null;
  change_type: ReconciliationChangeType;
  source_type: ReconciliationSourceType;
  section_reference: string;
  original_text: string;
  replacement_text: string;
}
```

**lib/inngest/functions/solicitation-reconciler.ts:**

Create an Inngest function triggered when all documents in a solicitation are classified.

```typescript
export const solicitationReconciler = inngest.createFunction(
  { id: 'solicitation-reconciler' },
  { event: 'solicitation.classification.complete' },
  async ({ event, step }) => { ... }
);
```

Steps:
1. `fetch-documents`: Query all solicitation_documents for this solicitation. Separate into: base documents (base_rfp, soo_sow_pws), amendments, Q&A responses, potential templates (pricing_template, cdrls, other_unclassified).

2. `reconcile-amendments`: If amendments exist and a base document exists:
   - Call reconcileAmendments(baseDoc, amendments)
   - Insert all ReconciliationResult records into document_reconciliations table
   - Update superseded solicitation_documents: set is_superseded=true, superseded_by=amendment_doc_id for docs that were fully superseded
   - Update amendment solicitation_documents: set processing_status='reconciled'

3. `parse-qa-modifications`: If Q&A response documents exist and a base document exists:
   - For each Q&A doc, call parseQAModifications(qaDoc, baseDoc)
   - Insert reconciliation records into document_reconciliations with source_type='qa_response'
   - Update Q&A solicitation_documents: set processing_status='reconciled'

4. `detect-templates`: For each potential template document:
   - Call detectTemplateFields(doc)
   - If fields returned, insert into template_field_mappings
   - Try to auto-fill values from company_profiles and solicitation data
   - Update solicitation_documents: set processing_status='reconciled'

5. `finalize`: Update all remaining solicitation_documents to processing_status='reconciled'. Update solicitation status to 'ready'. Log completion.

Error handling: If any step fails, log the error to processing_logs, mark the specific document as 'failed' with error_message, but continue processing other documents. Only set solicitation to 'failed' if ALL documents fail.

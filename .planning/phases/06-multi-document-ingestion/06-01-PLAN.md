  <name>Task 1: Create multi-document ingestion database migration</name>
  <files>supabase_tables/supabase-solicitation-ingestion-migration.sql</files>
  <action>
Create an additive migration SQL file with these tables:

1. **solicitations** table:
   - `id` UUID PK DEFAULT gen_random_uuid()
   - `company_id` UUID NOT NULL REFERENCES company_profiles(id)
   - `solicitation_number` TEXT NOT NULL (e.g., "W911NF-24-R-0001")
   - `title` TEXT (human-readable solicitation title)
   - `agency` TEXT (issuing agency name)
   - `status` TEXT NOT NULL DEFAULT 'uploading' CHECK IN ('uploading', 'classifying', 'reconciling', 'ready', 'failed')
   - `document_count` INTEGER DEFAULT 0
   - `created_at` TIMESTAMPTZ DEFAULT NOW()
   - `updated_at` TIMESTAMPTZ DEFAULT NOW()
   - UNIQUE constraint on (company_id, solicitation_number) to prevent duplicates

2. **solicitation_documents** table:
   - `id` UUID PK DEFAULT gen_random_uuid()
   - `solicitation_id` UUID NOT NULL REFERENCES solicitations(id) ON DELETE CASCADE
   - `filename` TEXT NOT NULL
   - `file_path` TEXT NOT NULL
   - `file_size` BIGINT (bytes)
   - `mime_type` TEXT
   - `document_type` TEXT CHECK IN ('base_rfp', 'soo_sow_pws', 'amendment', 'qa_response', 'pricing_template', 'dd254', 'clauses', 'cdrls', 'wage_determination', 'provisions', 'other_unclassified')
   - `document_type_label` TEXT (human-readable label for display, e.g., "Base RFP", "SOO/SOW/PWS")
   - `classification_confidence` TEXT CHECK IN ('high', 'medium', 'low') — only flag low-confidence per user decision
   - `classification_reasoning` TEXT (AI reasoning for classification)
   - `amendment_number` TEXT (for amendment docs, e.g., "Amendment 0001")
   - `effective_date` DATE (date the document was issued)
   - `is_superseded` BOOLEAN DEFAULT false (true if a later amendment supersedes this)
   - `superseded_by` UUID REFERENCES solicitation_documents(id) (which doc superseded this)
   - `extracted_text` TEXT (full text extracted from file)
   - `processing_status` TEXT NOT NULL DEFAULT 'pending' CHECK IN ('pending', 'extracting', 'classified', 'reconciled', 'failed')
   - `error_message` TEXT
   - `sort_order` INTEGER DEFAULT 0
   - `created_at` TIMESTAMPTZ DEFAULT NOW()
   - `updated_at` TIMESTAMPTZ DEFAULT NOW()

3. **document_reconciliations** table (tracks what changed between versions):
   - `id` UUID PK DEFAULT gen_random_uuid()
   - `solicitation_id` UUID NOT NULL REFERENCES solicitations(id) ON DELETE CASCADE
   - `source_document_id` UUID NOT NULL REFERENCES solicitation_documents(id) ON DELETE CASCADE (the amendment or Q&A that made the change)
   - `target_document_id` UUID REFERENCES solicitation_documents(id) ON DELETE SET NULL (the base doc or earlier amendment being modified)
   - `change_type` TEXT NOT NULL CHECK IN ('supersedes_section', 'modifies_scope', 'modifies_page_limit', 'modifies_eval_criteria', 'modifies_submission_instructions', 'adds_requirement', 'removes_requirement', 'general_modification')
   - `source_type` TEXT NOT NULL CHECK IN ('amendment', 'qa_response') — provenance tracking per user decision
   - `section_reference` TEXT (e.g., "Section L.4.2", "Question 15")
   - `original_text` TEXT (the text being superseded)
   - `replacement_text` TEXT (the new text from amendment/Q&A)
   - `is_active` BOOLEAN DEFAULT true (user can restore/undo reconciliation per user decision)
   - `created_at` TIMESTAMPTZ DEFAULT NOW()

4. **template_field_mappings** table (for fillable template detection):
   - `id` UUID PK DEFAULT gen_random_uuid()
   - `solicitation_document_id` UUID NOT NULL REFERENCES solicitation_documents(id) ON DELETE CASCADE
   - `field_name` TEXT NOT NULL
   - `field_type` TEXT CHECK IN ('text', 'number', 'date', 'boolean', 'selection')
   - `is_required` BOOLEAN DEFAULT true
   - `section_reference` TEXT
   - `auto_fill_source` TEXT (where this data might come from, e.g., "company_profiles.legal_name")
   - `auto_fill_value` TEXT (suggested pre-fill value)
   - `tag` TEXT CHECK IN ('action_required', 'reference_only') DEFAULT 'action_required' — per user decision
   - `created_at` TIMESTAMPTZ DEFAULT NOW()

Add indexes:
- solicitations(company_id)
- solicitations(solicitation_number)
- solicitation_documents(solicitation_id)
- solicitation_documents(document_type)
- solicitation_documents(processing_status)
- document_reconciliations(solicitation_id)
- document_reconciliations(source_document_id)
- template_field_mappings(solicitation_document_id)

Enable RLS on all tables. Add authenticated-user policies (same pattern as existing tables).

Use IF NOT EXISTS for all CREATE TABLE and CREATE INDEX statements so migration is safe to re-run.

Create three API route files following existing patterns (requireStaffOrResponse, X-Company-Id header, getServerClient):

**app/api/solicitations/route.ts:**
- `GET`: List solicitations for the company (from X-Company-Id header). Return array of solicitations ordered by created_at DESC. Include document_count.
- `POST`: Create a new solicitation. Accept `{ solicitation_number, title?, agency? }`. Validate solicitation_number is not empty. Check for duplicates (same company + solicitation_number). Return created solicitation with 201 status.

**app/api/solicitations/[id]/route.ts:**
- `GET`: Fetch single solicitation by ID with its documents. Join solicitation_documents ordered by sort_order, document_type. Return solicitation + documents array.
- `PATCH`: Update solicitation fields (title, agency, status). Return updated solicitation.

**app/api/solicitations/[id]/documents/route.ts:**
- `GET`: List all documents for a solicitation. Include classification data, processing status, amendment info. Order by: amendments first (by amendment_number), then other types by sort_order.

All endpoints:
- Use `requireStaffOrResponse()` for auth
- Get company_id from `X-Company-Id` header
- Verify the solicitation belongs to the requesting company before returning/modifying
- Return proper HTTP status codes (200, 201, 400, 403, 404, 500)
- Return `{ error: string }` on failure

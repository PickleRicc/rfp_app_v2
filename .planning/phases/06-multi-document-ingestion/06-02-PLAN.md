**app/api/solicitations/[id]/upload/route.ts:**

Create a POST endpoint that accepts multipart form data with multiple files. This is NOT the same as the existing /api/upload route (which processes single RFP documents for the v1 pipeline). This is a new endpoint specifically for solicitation package upload.

Implementation:
- Auth: requireStaffOrResponse() + X-Company-Id header
- Verify solicitation exists and belongs to requesting company
- Enforce Tier 1 gate (same pattern as existing upload route — check tier1_complete on company_profiles)
- Accept FormData with multiple files (field name: "files")
- Accept optional form field "solicitation_number_detected" for auto-detected solicitation number
- For each file:
  1. Validate file type — accept .pdf, .docx, .doc, .xlsx, .xls, .txt (reject others with error)
  2. Extract text content:
     - PDF: Send to PDF_SERVICE_URL/extract-text (same pattern as existing upload)
     - DOCX: Use file.text() for now (basic extraction — DOCX binary will need proper parsing later but text() gives XML content which AI can work with)
     - XLSX: Use file.text() (similarly, basic extraction)
     - TXT: Direct file.text()
  3. Create solicitation_documents record with: solicitation_id, filename, file_path (use /uploads/solicitations/{solicitation_id}/{filename}), file_size, mime_type, extracted_text, processing_status='extracting'
  4. Send Inngest event: 'solicitation.document.uploaded' with { solicitationId, documentId, extractedText }
- After all files processed, update solicitation document_count and set status='classifying'
- Return { success: true, documents: [{id, filename, status}...], errors: [{filename, error}...] }
- If some files fail and others succeed, continue processing valid files (per user decision: "failures are flagged clearly but processing continues for valid files")

**lib/inngest/functions/solicitation-classifier.ts:**

Create a new Inngest function that classifies solicitation documents by type.

```typescript
export const solicitationDocumentClassifier = inngest.createFunction(
  { id: 'solicitation-document-classifier' },
  { event: 'solicitation.document.uploaded' },
  async ({ event, step }) => { ... }
);
```

Steps:
1. `log-start`: Update solicitation_documents processing_status to 'extracting', log to processing_logs
2. `classify-document`: Call Anthropic with the extracted text (first 15000 chars). Prompt should ask Claude to:
   - Classify into one of the 11 document types: base_rfp, soo_sow_pws, amendment, qa_response, pricing_template, dd254, clauses, cdrls, wage_determination, provisions, other_unclassified
   - Extract amendment_number if applicable (e.g., "Amendment 0003")
   - Extract effective_date if visible
   - Extract solicitation_number if visible in the document
   - Provide confidence: high/medium/low
   - Provide brief reasoning
   - Return JSON: { document_type, document_type_label, amendment_number, effective_date, solicitation_number_detected, confidence, reasoning }
3. `update-classification`: Update solicitation_documents record with: document_type, document_type_label, classification_confidence, classification_reasoning, amendment_number, effective_date, processing_status='classified'
4. `log-completion`: Log to processing_logs
5. `check-all-classified`: Query all solicitation_documents for this solicitation. If ALL have processing_status='classified', update solicitation status to 'reconciling' and send event 'solicitation.classification.complete' with { solicitationId }. This triggers Plan 03's reconciliation logic.

Use the extractJSON helper pattern from the existing stage0-classifier.ts.

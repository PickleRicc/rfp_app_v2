  <name>Task 1: ComplianceExtractionView component with accordions, inline editing, and amendment impact</name>
  <files>
    app/components/solicitation/ComplianceExtractionView.tsx
  </files>
  <action>
    Create `app/components/solicitation/ComplianceExtractionView.tsx` — a 'use client' component.

    **Props:**
    ```typescript
    interface ComplianceExtractionViewProps {
      solicitationId: string;
      companyId: string;
    }
    ```

    **Data fetching:**
    - On mount, fetch `GET /api/solicitations/{solicitationId}/compliance` with X-Company-Id header
    - Store extractions in state grouped by category (Record<ExtractionCategory, ComplianceExtraction[]>)
    - Poll every 5 seconds while any extraction has extraction_status='pending' or 'extracting' (same polling pattern as SolicitationUploader from Phase 6 — 5s interval, max 60 attempts = 5 min)
    - Stop polling when all extractions have 'completed' or 'failed' status

    **Layout — Accordion sections (per user decision):**
    All 4 categories rendered as collapsible accordion sections. Default: all expanded on first load (per user decision in CONTEXT.md specifics). Each accordion has:

    1. **Header bar** — Category icon (from EXTRACTION_CATEGORY_ICONS), category label (from EXTRACTION_CATEGORY_LABELS), extraction status indicator (spinner if extracting, checkmark if completed, X if failed), "Re-extract" button (right-aligned, per user decision)
    2. **Content area** — Category-specific rendering (see below)
    3. **Collapse/expand** — Click header to toggle, ChevronDown/ChevronUp icon

    **Category-specific rendering:**

    **Section L (EXTRACT-01):**
    - Volume structure: table with columns — Volume Name, Page Limit, Format/Rules. Each cell editable on click.
    - Required attachments: bulleted list, each item editable
    - Required forms/certifications: bulleted list, each item editable
    - Formatting rules: key-value display (Font, Size, Margins, Spacing, Other), each value editable

    **Section M (EXTRACT-02):**
    - Evaluation factors: indented tree structure per user decision
      - Top-level factors as cards/rows with factor name + weight description badge
      - Weight badges colored: "Most Important" or "Significantly More Important" in green, "Equal" or "Approximately Equal" in blue, "Less Important" in gray (per user decision)
      - Subfactors indented below parent factors with name + description
    - Evaluation methodology: single value display (LPTA / Best Value / Tradeoff)
    - Experience thresholds: bulleted list
    - Key personnel requirements: table with Role, Qualifications columns

    **Admin Data (EXTRACT-03):**
    - Key-value pairs display (Claude's discretion format per user decision): NAICS Code, Size Standard, Set-Aside Designation, Contract Type, Period of Performance
    - CLIN Structure: table with CLIN Number, Description, Type columns

    **Rating Scales (EXTRACT-04):**
    - Per-factor rating scale table: Factor Name header, then Rating | Description rows
    - Factor weightings: table with Factor Name, Weight/Importance columns

    **Confidence indicators (per user decision):**
    - Each extracted value wrapped in a container with border color based on confidence:
      - high: `border-l-4 border-green-400` (green left border)
      - medium: `border-l-4 border-amber-400` (amber left border)
      - low: `border-l-4 border-red-400` (red left border)
    - Match Phase 6 classification confidence visual language (per CONTEXT.md specifics)

    **Inline editing (per user decision):**
    - Click any extracted value to switch to edit mode (input/textarea replaces display text)
    - Save on Enter or blur; cancel on Escape
    - On save: PATCH `/api/solicitations/{id}/compliance` with `{ extraction_id, field_value }`
    - After save: value shows "User Override" badge (small purple pill badge, e.g., `bg-purple-100 text-purple-700 text-xs`)
    - Already-overridden values always show the badge
    - Overridden values have a small "Revert" link/button next to the badge — calls PATCH with `{ extraction_id, action: 'revert' }`

    **Missing values (per user decision):**
    - When field_value is null or empty: show `<span class="text-gray-400 italic">Not found</span>`
    - Clicking "Not found" opens inline edit mode so user can manually enter a value

    **Re-extract (per user decision):**
    - "Re-extract" button on each accordion header
    - On click: if any extractions in that category have is_user_override=true, show a confirmation prompt: "You have manually edited X values in this section. Re-extraction will update AI-extracted values but preserve your overrides. You'll be prompted to review any conflicts. Continue?"
    - On confirm: PATCH `/api/solicitations/{id}/compliance` with `{ category, action: 're-extract' }`
    - After re-extract triggers: show extracting spinner on that section, poll for completion

    **Amendment impact display (per user decision):**
    - When an extraction has `amendment_source_id` set and `amendment_note` is not null:
      - Show the original value with strikethrough (match ReconciliationView.tsx strikethrough pattern)
      - Show the new value below/beside it
      - Show "Amendment X" source badge (amber pill, matching ReconciliationView source badge pattern from Phase 6)
    - When is_user_override=true AND amendment_source_id is set:
      - Show conflict indicator: "You edited this — amendment may have changed it. Review?" (per user decision)
      - Provide "Accept new value" and "Keep my override" buttons

    **Skeleton loaders (per user decision):**
    - While extraction_status is 'pending' or 'extracting' for a category:
      - Show accordion header (with category name and spinner)
      - Content area shows animated skeleton blocks matching the expected layout (rows of gray bars with pulse animation)
    - Once extraction completes: replace skeleton with real data

    **Error states:**
    - When extraction_status is 'failed' for a category:
      - Show empty section with message: "Extraction failed — {error_message}" if available, else "No data found — this document may not contain {category_label} content"
      - Show "Retry" button (triggers re-extract for that category)
      - Show "Add manually" option (opens inline edit for empty fields)

    **Empty state (no extractions at all):**
    - If no compliance_extractions rows exist for the solicitation:
      - Show message: "Compliance extraction has not run yet. It will start automatically after document reconciliation completes."
      - If solicitation status is 'ready': show "Extract Now" button that triggers extraction via the API
  </action>

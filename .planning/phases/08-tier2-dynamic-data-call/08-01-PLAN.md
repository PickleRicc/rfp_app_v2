Create additive SQL migration `supabase-tier2-data-call-migration.sql` with two tables:

**`data_call_responses`** — stores one row per solicitation's data call state:
- `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
- `solicitation_id` UUID NOT NULL REFERENCES solicitations(id) ON DELETE CASCADE
- `company_id` UUID NOT NULL
- `status` TEXT NOT NULL DEFAULT 'in_progress' CHECK (status IN ('in_progress', 'completed'))
- `opportunity_details` JSONB DEFAULT '{}' — prime/sub role, teaming partners, contract type, NAICS size
- `past_performance` JSONB DEFAULT '[]' — array of past performance reference objects
- `key_personnel` JSONB DEFAULT '[]' — array of key personnel objects with file references
- `technical_approach` JSONB DEFAULT '{}' — summary, tools, staffing model, assumptions, risks
- `compliance_verification` JSONB DEFAULT '{}' — org certs, individual certs, facility clearance, NIST score
- `form_schema` JSONB — the generated form schema (cached for fast reload)
- `completed_at` TIMESTAMPTZ
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- `updated_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- UNIQUE(solicitation_id, company_id) — one data call response per solicitation per company
- Index on (company_id, solicitation_id)
- RLS: enable, permissive policy for authenticated users (same pattern as compliance_extractions)

**`data_call_files`** — stores uploaded files (resumes, certs, LOCs, attachments):
- `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
- `data_call_response_id` UUID NOT NULL REFERENCES data_call_responses(id) ON DELETE CASCADE
- `section` TEXT NOT NULL CHECK (section IN ('past_performance', 'key_personnel', 'compliance_verification'))
- `field_key` TEXT NOT NULL — identifies which specific field this file belongs to (e.g., 'personnel_0_resume', 'org_cert_iso')
- `filename` TEXT NOT NULL
- `file_path` TEXT NOT NULL — Supabase storage path
- `file_size` INTEGER
- `mime_type` TEXT
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- Index on (data_call_response_id, section)
- RLS: enable, permissive policy for authenticated users

Use IF NOT EXISTS pattern for both tables (additive migration safe to re-run).

Create `lib/ingestion/data-call-generator.ts` — the core form schema generator:

**`generateDataCallSchema(solicitationId: string)`** function:
1. Query `compliance_extractions` for the given solicitation_id, grouped by category
2. Build 5 `DataCallSection` objects:

**Section 1: Opportunity Details** (id: 'opportunity_details')
- Fields: prime_or_sub (select: prime/sub/teaming), teaming_partners (textarea), contract_type (text, pre-filled from admin_data extraction if found), naics_code (text, pre-filled from admin_data), size_standard (text, pre-filled from admin_data), set_aside (text, pre-filled from admin_data)
- All fields required except teaming_partners
- rfp_citation: pull from the corresponding admin_data extraction's source info

**Section 2: Past Performance** (id: 'past_performance')
- Read `past_performance` category extractions for references_required count
- If references_required found and is a number, use it as dynamic_count; otherwise default to 3
- dynamic_count_source: the rfp_citation text (e.g., "Per Section L.4.3: 3 references required")
- Fields per reference card: project_name (text, required), client_agency (text, required), contract_number (text, required), contract_value (text, optional), period_of_performance (text, required), relevance_summary (textarea, required), contact_name (text, required), contact_email (text, required), contact_phone (text, optional)
- Also read recency_requirement and relevance_criteria from extraction and include as section description text

**Section 3: Key Personnel** (id: 'key_personnel')
- Read `key_personnel` category extractions for positions array
- Generate one card per position, with role pre-filled from extraction
- Fields per card: role (text, pre-filled, required), name (text, required), qualifications_summary (textarea, required), resume_file (file, required), certifications_files (file, optional), loc_file (file, conditional based on extraction loc_requirements)
- dynamic_count from positions array length, or default to 0 if no key personnel extracted
- rfp_citation from key_personnel extraction source

**Section 4: Technical Approach** (id: 'technical_approach')
- Fields: approach_summary (textarea, required), tools_and_platforms (textarea, optional), staffing_model (textarea, optional), assumptions (textarea, optional), risks (textarea, optional)
- rfp_citation: pull from sow_pws extraction task_areas summary if available
- All static fields (no dynamic count)

**Section 5: Compliance Verification** (id: 'compliance_verification')
- Read `security_reqs` extractions for clearance levels, CMMC, NIST requirements
- Read `section_l` extractions for required forms/certifications
- Fields: org_certifications (boolean toggles for each cert found in extraction), individual_certifications (textarea), facility_clearance_confirmed (boolean, required if dd254_required), nist_800_171_score (text, required if nist_800_171_required), required_attachments (file per required attachment from section_l)
- rfp_citation for each field from corresponding extraction source

For each field, set `rfp_citation` to a short reference string like "Per Section L.4.2" or "Per Section M.3" — derived from the compliance_extraction's `field_label` and category. If no specific citation is available, set to null.

Helper: `getExtractionValue(extractions, category, fieldName)` — finds the extraction row and returns its field_value.

Create `app/api/solicitations/[id]/data-call/route.ts`:

**GET**:
- Auth: requireStaffOrResponse() + X-Company-Id header + solicitation ownership check (same pattern as compliance/route.ts)
- Async params pattern for Next.js 15+ (await params)
- Call generateDataCallSchema(solicitationId) to get form schema
- Query data_call_responses for existing saved data (if any)
- Query data_call_files for existing uploaded files (if any)
- Return { schema: DataCallFormSchema, response: DataCallResponse | null, files: DataCallFile[] }

**PUT**:
- Auth: same as GET
- Body: { opportunity_details?, past_performance?, key_personnel?, technical_approach?, compliance_verification?, status? }
- Upsert into data_call_responses (ON CONFLICT solicitation_id, company_id DO UPDATE)
- If status = 'completed', set completed_at = now()
- Return updated response

Create `app/api/solicitations/[id]/data-call/upload/route.ts`:

**POST**:
- Auth: same pattern
- Accept multipart/form-data with: file (File), section (string), field_key (string)
- Validate file size (max 10MB), file type (PDF, DOCX, DOC, PNG, JPG, JPEG)
- Upload to Supabase storage bucket 'data-call-files' at path: `{company_id}/{solicitation_id}/{section}/{field_key}/{filename}`
- Create data_call_files record
- Return { file: DataCallFile }

**DELETE** (in same route file):
- Auth: same pattern
- Body: { file_id: string }
- Delete from Supabase storage and data_call_files table
- Return { success: true }

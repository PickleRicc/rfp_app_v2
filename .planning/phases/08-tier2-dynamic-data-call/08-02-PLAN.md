Create `app/components/datacall/DataCallView.tsx` — the main Data Call container:

**Props:** `{ solicitationId: string, companyId: string }`

**State:**
- `schema: DataCallFormSchema | null` — the generated form schema from API
- `formData: Record<string, unknown>` — the current form data (keyed by section id)
- `savedData: Record<string, unknown>` — last-saved data (for dirty detection)
- `openSections: Set<string>` — which accordion sections are currently open
- `loading: boolean` — initial load state
- `saving: boolean` — save-in-progress state
- `error: string | null`

**On mount:** Fetch GET /api/solicitations/{id}/data-call with X-Company-Id header. Set schema, initialize formData from response (or empty defaults), set savedData to same.

**Accordion layout:**
- Render each section from schema.sections as a collapsible panel
- Section header: icon (from a static map by section id), title, description summary, green checkmark (CheckCircle2 from lucide) when isSectionComplete(sectionId)
- Click header to toggle open/close. Multiple can be open simultaneously (openSections is a Set, not single active).
- Smooth height transition (CSS transition on max-height or use a simple open/closed with overflow-hidden + transition-all pattern). Keep it simple — no animation library.

**Section completion check (`isSectionComplete`):**
- For each section, check that all fields with `required: true` in the schema have non-empty values in formData
- For dynamic-count sections (past_performance, key_personnel), check that each card's required fields are filled

**Save Progress button:**
- Fixed at bottom of the Data Call area (not fixed to viewport — just below the last section)
- Calls PUT /api/solicitations/{id}/data-call with current formData
- Shows "Saving..." state during request
- On success, update savedData to match formData (clears dirty state)
- Disabled when !isDirty (formData === savedData deep comparison, use JSON.stringify comparison)

**Dirty detection + beforeunload:**
- `isDirty` computed from JSON.stringify(formData) !== JSON.stringify(savedData)
- Register window.addEventListener('beforeunload') that calls e.preventDefault() when isDirty
- Clean up listener on unmount

**Section rendering:**
- Route each section to the appropriate component based on section.id:
  - 'opportunity_details' -> OpportunityDetailsSection
  - 'past_performance' -> PastPerformanceSection (Plan 02 Task 2)
  - 'technical_approach' -> TechnicalApproachSection (Plan 02 Task 2)
  - 'key_personnel' -> KeyPersonnelSection (Plan 03 — render placeholder div for now)
  - 'compliance_verification' -> ComplianceVerificationSection (Plan 03 — render placeholder div for now)

For Plan 03 sections, render a simple placeholder: `<div className="p-4 text-sm text-muted-foreground">Section coming in next plan...</div>`

**Error state:** If schema fetch fails, show an error card with retry button (same pattern as ComplianceExtractionView).

**Loading state:** Skeleton loader while schema loads (same pattern as ComplianceExtractionView).

- onChange triggers on any field change, passing updated OpportunityDetails to parent

**Styling:** Follow Tier 1 tab form patterns — labeled inputs with gap-2, border-border inputs, focus:ring-2.

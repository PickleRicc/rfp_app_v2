Create `app/components/datacall/KeyPersonnelSection.tsx`:

**Props:** `{ section: DataCallSection, data: KeyPersonnelEntry[], onChange: (data: KeyPersonnelEntry[]) => void, files: DataCallFile[], solicitationId: string, companyId: string, onFileUploaded: (file: DataCallFile) => void, onFileRemoved: (fileId: string) => void }`

**Dynamic card rendering (same pattern as PastPerformanceSection):**
- Read `section.dynamic_count` for number of key personnel cards
- Each card header: position role (pre-filled from extraction) + card number
- If dynamic_count is 0 (no key personnel extracted), show a message: "No key personnel requirements extracted from the RFP. Add positions manually if needed."
- Add/Remove position buttons (same pattern as past performance cards)

**Card fields per KeyPersonnelEntry:**
- role: text input (required, pre-filled from extraction position title)
- name: text input (required)
- qualifications_summary: textarea (required) — placeholder hinting at extraction qualifications if available
- resume_file: inline file upload button (required per CONTEXT.md)
- certifications_files: inline file upload button (optional, multiple allowed)
- loc_file: inline file upload button (conditional — shown only if section has LOC requirement from extraction)

**Inline file upload pattern (per CONTEXT.md decisions):**
- Each file field has a small "Browse..." button (styled as secondary button, no drag-drop)
- On click: opens hidden `<input type="file" accept=".pdf,.docx,.doc,.png,.jpg,.jpeg">` via ref.click()
- On file selected: POST to /api/solicitations/{solicitationId}/data-call/upload with FormData containing: file, section='key_personnel', field_key=`personnel_{index}_{fieldType}` (e.g., 'personnel_0_resume')
- During upload: show small spinner next to button
- After upload: display `filename` text + "Remove" button (red text, no icon). No thumbnails or previews per CONTEXT.md.
- Remove button: calls DELETE /api/solicitations/{solicitationId}/data-call/upload with { file_id } in body, then calls onFileRemoved
- File size limit: 10MB. Type whitelist: PDF, DOCX, DOC, PNG, JPG, JPEG. Show error message below button if file exceeds limit or wrong type (client-side check before upload).

**Map files to fields:** Use the `files` prop to match existing uploaded files to each card's fields via the field_key pattern.

**File state management:**
- Add `files: DataCallFile[]` to state, loaded from GET API response
- `onFileUploaded(file: DataCallFile)`: append to files state
- `onFileRemoved(fileId: string)`: remove from files state

**"Complete Data Call" button:**
- Positioned below the Save Progress button (or replaces it when all sections show complete)
- Per CONTEXT.md: "Field validation runs on submit (when user clicks 'Complete Data Call'), not inline"
- On click, run validation across all sections:

**Validation logic (`validateDataCall`):**
- For each section in schema.sections:
  - For each field with required=true: check that the corresponding value in formData is non-empty
  - For dynamic-count sections (past_performance, key_personnel): iterate each card, check required fields per card
  - For file fields: check that a matching file exists in the files array
- Collect all errors as `{ sectionId: string, fieldKey: string, message: string }[]`
- If errors exist:
  - Show error summary at top of Data Call view: "X fields in Y sections need attention"
  - Auto-expand sections that have errors
  - Scroll to top of Data Call view
  - Each section header shows a red warning icon instead of green checkmark when it has validation errors
  - Do NOT block Save Progress — only block Complete Data Call
- If no errors:
  - Call PUT API with status='completed'
  - Show a success confirmation: green banner "Data Call Complete — Ready for Draft Generation"
  - Disable all form fields and file uploads (read-only mode)
  - Change "Complete Data Call" button to "Reopen Data Call" which sets status back to 'in_progress'

**Section completion indicators (update existing logic):**
- Green CheckCircle2 icon on section header when all required fields filled (and required files uploaded)
- Red AlertCircle icon on section header when validation has been run and this section has errors
- No icon when section is partially filled and validation hasn't been triggered yet

**Loading states:**
- File upload in progress: small spinner next to the file field being uploaded
- Save in progress: "Saving..." text on Save Progress button
- Complete in progress: "Completing..." text on Complete Data Call button with disabled state

**Summary panel (Claude's discretion per CONTEXT.md):**
- Add a small summary at the top of the Data Call view (above the accordion) showing:
  - Status: "In Progress" or "Completed" badge
  - Sections: "3 of 5 complete" or similar count
  - Last saved: timestamp of last save (from response.updated_at)
- This helps user gauge overall progress without expanding each section

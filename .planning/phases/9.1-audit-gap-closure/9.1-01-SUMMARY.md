### Task 2: field_label in Draft Generator SELECT

Fixed `lib/inngest/functions/proposal-draft-generator.ts`:

- **Line 286 (before):** `.select('category, field_name, field_value, confidence, is_user_override')`
- **Line 286 (after):** `.select('category, field_name, field_label, field_value, confidence, is_user_override')`
- **Line 311 (before):** `field_label: row.field_name,`
- **Line 311 (after):** `field_label: row.field_label || row.field_name,`

The draft generator assembles ComplianceExtraction objects to feed into the prompt assembler. Previously, it used field_name (machine identifier like `"volume_structure"`) as the field_label. Now it reads the actual human-readable label from the database (e.g., `"Volume Structure (Section L.3.2)"`), which improves DRAFT-02 prompt quality for human reviewers. The fallback to field_name handles any rows where field_label is null.

### Task 3: TIER1_INCOMPLETE 403 Clickable Profile Link

Fixed `app/components/solicitation/SolicitationUploader.tsx`:

- Added `uploadError` state typed as `ReactNode | null` (line 59) to support JSX-based error messages
- Added TIER1_INCOMPLETE check before generic error fallback in the `if (!response.ok)` block — detects `response.status === 403 && data.code === "TIER1_INCOMPLETE"` and sets a clickable error with an `<a href="/company/profile">` link
- Added red alert box in JSX to render `uploadError` when set
- Generic string error fallback for all other non-ok responses unchanged

Before: Users seeing a TIER1_INCOMPLETE 403 got a plain error string with no navigation path.
After: Users see "Your company profile is incomplete. [Complete your company profile] before uploading solicitation documents." with a clickable link.

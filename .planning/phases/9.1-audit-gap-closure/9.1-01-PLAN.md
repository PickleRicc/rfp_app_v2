Fix the audit-identified UX gap in `app/components/solicitation/SolicitationUploader.tsx` lines 222-231.

**Current state (lines 222-231):**
The error handler treats ALL non-ok responses identically — sets a generic error string from `data.error`. When the server returns a 403 with `data.code === 'TIER1_INCOMPLETE'`, the user sees the error text but has no clickable way to navigate to `/company/profile` to fix it.

**Fix:**

In the `if (!response.ok)` block (line 222), BEFORE the generic error fallback, check for the TIER1_INCOMPLETE case specifically:

```typescript
if (!response.ok) {
  // Check for Tier 1 incomplete gate
  if (response.status === 403 && data.code === 'TIER1_INCOMPLETE') {
    setFiles((prev) =>
      prev.map((f) => ({
        ...f,
        status: "failed" as FileUploadStatus,
        error: "Company profile incomplete",
      }))
    );
    // Set a component-level error with navigation instruction
    setError(
      <span>
        Your company profile is incomplete.{' '}
        <a
          href="/company/profile"
          className="text-blue-600 underline hover:text-blue-800"
        >
          Complete your company profile
        </a>{' '}
        before uploading solicitation documents.
      </span>
    );
    return;
  }

  // Generic error fallback (existing code)
  setFiles((prev) =>
    prev.map((f) => ({
      ...f,
      status: "failed" as FileUploadStatus,
      error: data.error || "Upload failed",
    }))
  );
  return;
}
```

**Important implementation notes:**

1. Check if `setError` already exists on the component. If not, you need to add an `error` state variable: `const [error, setError] = useState<React.ReactNode | null>(null);`
2. Render the `error` state in the component's JSX (a red alert box above or below the upload area).
3. If the component already has an error state mechanism, use it. If it uses string-only errors, change the type to `React.ReactNode` to support JSX with the clickable link.
4. The `<a>` tag uses standard Next.js-compatible href — no client-side router import needed for a simple link.
5. Use `import Link from 'next/link'` if a `Link` component is already imported in the file; otherwise a plain `<a>` tag is fine for this navigation.

Read the full component file first to understand its existing error handling pattern before making changes.
